## 创建型模式

| 模式 | 描述 | 状态 |
| -- | -- | -- |
| 单例模式（Singleton） | 确保一个类只有一个实例，并提供一个全局访问点。通常通过私有构造方法、静态实例和静态方法实现，适用于需要唯一实例的场景，如配置管理器或线程池。 | ✅ |
| 简单工厂（Simple Factory） | 通过一个工厂类根据参数创建不同类型的对象，集中管理对象创建逻辑。客户端只需调用工厂方法，无需了解具体类，但不符合开闭原则。 | ✅ |
| 工厂方法（Factory Method） | 定义一个创建对象的接口，由子类实现具体对象的创建。延迟对象创建到子类，解耦创建与使用，符合开闭原则，适合动态决定对象类型的场景。 | ✅ |
